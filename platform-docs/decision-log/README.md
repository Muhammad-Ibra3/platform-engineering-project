# Decision Log

This directory captures architectural decisions and why they were made.

## Files

- `decisions.md` - Current adopted decisions.
- `adr-template.md` - Template for adding a new decision record.

## When to add an entry

Add a decision whenever a change affects one or more of:

- platform topology or component selection
- security model or policy enforcement mode
- repository structure and path conventions
- deployment orchestration (sync waves, prune strategy)
- CI/GitOps contract and automation behavior
